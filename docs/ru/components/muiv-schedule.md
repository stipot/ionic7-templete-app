# Расписание MUIV

> Компонент отображает актуальное расписание занятий вуза МУИВ (muiv.ru). Пользователь выбирает факультет, курс и группу, после чего загружается расписание с внешнего API.

Маршрут приложения: `/schedule` .

---

## API

По умолчанию используется базовый URL: `https://muiv2.vless-service.digital`. Его можно изменить через меню (кнопка в шапке hero-блока).

| Эндпоинт | Метод | Назначение |
|----------|-------|------------|
| `{baseUrl}/structure` | GET | Иерархия факультетов → курсов → групп |
| `{baseUrl}/schedule` | GET | Расписание для выбранной группы |

Параметры запроса `/schedule`:

- `faculty` — название факультета
- `course` — курс
- `group` — учебная группа

---

## Основные типы

### `Lesson`

Занятие в расписании дня.

| Поле | Тип | Описание |
|------|-----|----------|
| `pairNumber` | `number` | Номер пары |
| `subject` | `string` | Название дисциплины |
| `timeStart` | `string` | Время начала |
| `timeEnd` | `string` | Время окончания |
| `location` | `string` | Аудитория / место |
| `type` | `string` | Тип занятия: «Лекция», «СПЗ», «Консультация» и др. |

### `ScheduleDay`

Один день в массиве расписания.

| Поле | Тип | Описание |
|------|-----|----------|
| `date` | `string` | Дата (отображается во вкладке дня) |
| `dayLabel` | `string` | Название дня (например, «Пн») |
| `lessons` | `Lesson[]` | Список занятий на этот день |


### `SavedSelection`

Сохранённый в `localStorage` выбор пользователя (ключ `muiv_schedule_selection`).

| Поле | Тип | Описание |
|------|-----|----------|
| `faculty` | `string \| null` | Факультет |
| `course` | `string \| null` | Курс |
| `group` | `string \| null` | Группа |

## Возможные доработки

1. **Обработка ошибок API** - можно показывать пользователю toast или блок с текстом при сбое `/structure` или `/schedule`, а не только скрывать загрузку.
2. **Pull-to-refresh** - обновление расписания жестом в `ion-content`.
3. **Кэш** - сохранять последний успешный ответ `/schedule` в `localStorage` или IndexedDB для просмотра без сети.
4. **Уведомления** - напоминание о ближайшей паре (Capacitor Local Notifications).
5. **Профиль** - добавить возможность создавать заготовленные профили, с сохранением в аккаунт(соответственно нужно сделать функциональность аккаунтов)
